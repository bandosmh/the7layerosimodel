Layer 7

Application Layer

Application Layer

Serves as the interface between the network and user applications.

Common Examples:

  • When a person types a website in a browser, it uses HTTP to request and display the page.
  • Online banking accounts use HTTPS to protect sensitive information.
  • A web developer uploading website files from their computer to a web server using an FTP client like FileZilla.
  • When you send an email through Gmail or Outlook, it's sent by using SMTP servers.
  • When people type "youtube.com", DNS finds its correct IP address so the browser can connect.

Detailed Use Cases:

  • When you open Chrome, Safari, or Edge and go to a site like https://amazon.com, HTTPS is used to securely load the web page.
  • When you hit "Send" in Gmail or Outlook, your email is pushed to the mail server using SMTP.
  • When you open your inbox in the Gmail app, it uses IMAP to sync and fetch your emails from the server.
  • A graphic designer might upload a large file to a company's FTP server so coworkers can download it. Developers also use FTP to upload website code.
  • When you type netflix.com or open the TikTok app, DNS translates that name into the server's IP address so your device knows where to connect.

The application layer interacts with the lower layers to move data generated by the end user, like an email, across the network.

Application layer is important because it has the secure shell network protocol, which in return helps users transfer files between computers securely.

Layer 6

Presentation Layer

Presentation Layer Overview

Acts as a translator between applications and the network, ensuring data compatibility between different systems.

Key Functions:

  • Data Translation: Converts data formats between sender and receiver for mutual understanding
  • Encryption/Decryption: Secures data transmission using protocols like SSL/TLS
  • Compression/Decompression: Reduces file sizes for efficient transfer

Common Protocols & Formats:

  • JPEG: Image compression
  • MP3: Audio compression
  • ASCII: Standard character encoding
  • SSL/TLS: Secure communication protocols

Important Concepts:

  • Functions as a "universal translator" for different data formats
  • Prevents miscommunication between systems using different encoding methods
  • Bridges the gap between human-facing applications and machine-readable data

Layer 6 prepares data from applications by formatting, encrypting, or compressing it for network transmission, then reverses these processes for incoming data.
