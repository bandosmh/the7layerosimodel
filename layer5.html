Session Layer (Layer 5)

Definition

Layer 5 of the OSI model, called the Session Layer, is responsible for establishing, managing, and canceling communication sessions between two applications.

Session Setup

  • Initiates a connection between applications on different devices.
  • Establishes session parameters like authentication, authorization, and connection settings.

Session Maintenance

  • Manages data exchange during communication.
  • Keeps track of which side is sending or receiving at any given time (dialog control).
  • Handles checkpoints or synchronization points so that if a failure occurs, data transfer can resume instead of restarting.

Session Termination

Closes sessions once communication is complete, ensuring all data has been properly transmitted.

Synchronization

Inserts “checkpoints” into long data transfers (such as large file transfers or video streams), allowing resumption from the last checkpoint if interrupted.

Example Protocols

  • NetBIOS — Manages session communication and name registration for networked devices, especially in older Windows networks.
  • SCP (Session Control Protocol) — Used in DECnet for managing and controlling sessions between nodes.
  • ASP (AppleTalk Session Protocol) — Provides session services for AppleTalk networks, handling session establishment and termination.
  • SOCKS5 — Manages session control for proxy connections, authenticating and maintaining data flow between client and server.

Ongoing Communications (e.g., video calls)

  • Keeps the session open and synchronized as data continuously flows.
  • Detects interruptions or idle periods and can reestablish sessions automatically.
  • Ensures both devices remain in agreement about the session’s state (for example, whether it’s paused, active, or ended).
  • Uses synchronization points to resume communication from a specific point if the connection temporarily fails.

Relationship to Other Layers

  • With the Presentation Layer (Layer 6): Prepares and hands off formatted, encrypted, or compressed data so the Session Layer can manage the session carrying that data.
  • With the Transport Layer (Layer 4): Relies on Transport services (like reliable delivery) to move the session’s data between endpoints.

Summary

The Session Layer manages the connection between two computers or devices. It starts and keeps the connection open while they share data, ensures the connection stays stable during things like video calls or online chats, and closes the connection properly when the exchange is done.
