Transport Layer (Layer 4)

Definition

Layer 4 of the OSI model, known as the Transport Layer, is responsible for the end to end delivery of data between devices. It ensures that data sent from one application on a device is properly received and assembled by the correct application on another device.

The Transport Layer manages how much data is sent, how fast it’s sent, and verifies that it arrives correctly and in order.

Core Functions

  • Dividing and Reassembly: Large data streams from higher layers are divided (segmented) into smaller packets for transmission and reassembled on the receiving end.
  • Error Checking: Uses checksums or sequence numbers to detect errors or lost packets and requests retransmission if needed (in reliable protocols like TCP).
  • Flow Control: Manages the rate of data transmission to prevent overwhelming the receiver.
  • Connection Control: Can operate in connection-oriented (reliable) or connectionless (fast, best-effort) modes and is responsible for logically setting up and tearing down connections.

Examples of Protocols

TCP (Transmission Control Protocol)

  • Reliable and connection-based
  • Ensures all packets arrive correctly and in sequence
  • Performs error recovery, retransmission, and flow control
  • Used for applications where accuracy matters more than speed

UDP (User Datagram Protocol)

  • Unreliable and connectionless
  • Sends packets without establishing a dedicated connection
  • Faster, but there is no guarantee of delivery
  • Used for time-sensitive applications

Real-World Examples

  • Web Browsing (HTTP over TCP): TCP ensures every byte of data (HTML, images, CSS) delivers accurately and in order.
  • Video Streaming or Gaming (UDP): UDP sends data faster by skipping retransmissions; occasional packet loss is tolerated to maintain real-time performance.

Relationship to Other Layers

  • With Layer 3 (Network Layer): The Transport Layer uses the Network Layer to route packets between devices across networks. The Network Layer handles where the data goes; the Transport Layer manages how it gets there reliably.
  • With Layer 5 (Session Layer): The Transport Layer provides reliable data transfer services that the Session Layer uses to manage ongoing communication sessions.

Summary

The Transport Layer ensures data moves safely and correctly between devices: it breaks large amounts of data into smaller parts, reassembles them on receipt, checks for errors, and controls transmission speed. TCP and UDP are the most common transport protocols, offering trade-offs between reliability and speed.
